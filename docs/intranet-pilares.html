    Pilar 01

    Instructions

    Regras permanentes injetadas em todo contexto do Copilot — automaticamente, sem ação do dev.

    • Sempre ativo
    • HTML semântico
    • WCAG AA
    • SEO
    # .github/copilot-instructions.md
## HTML Semântico
- Use <section>, <article>, <header>
- Nunca use <div> onde semântico se aplica
- Classes CSS: prefixo fc-

    Arquivos no repositório

    • .github/copilot-instructions.md — regras globais do time
    • .github/instructions/acessibilidade.instructions.md — WCAG AA
    • .github/instructions/seo.instructions.md — meta tags, OG, alt

    O que isso garante

    • Toda imagem gerada terá alt descritivo
    • Botões e links terão texto visível ou aria-label
    • Classes CSS sempre com prefixo fc-
    • Hierarquia h1 → h2 → h3 nunca pulando nível
  • 🧠
    Pilar 02

    Skills

    Conhecimento especializado injetado sob demanda — o Copilot decide quando o contexto é relevante.

    • Sob demanda
    • Templates
    • Mobile-first
    # .github/skills/gerar-secao/SKILL.md
## Quando usar: seção, hero, cards,
  planos, depoimentos, rodapé

## Template
<section aria-labelledby="id">
  /* mobile-first + fc- prefix */

    Skills disponíveis

    • .github/skills/gerar-secao/ — template de seção HTML
    • .github/skills/precos/ — template de card de planos
    • .github/skills/suporte/ — template de FAQ / suporte

    Skills vs. Instructions

    • Instructions = regras que sempre valem ("imagem precisa de alt")
    • Skills = procedimentos pontuais ("ao gerar seção, use este template")
    • Skill entra quando o Copilot detecta o contexto adequado
  • 🤖
    Pilar 03

    Agents

    Agentes com identidade, escopo e ferramentas próprias. Invocados explicitamente com @nome.

    • Invocação explícita
    • Ferramentas próprias
    • QA automatizado
    # Como invocar:
@qa-academia revise o index.html
@qa-academia verifique acessibilidade

# O agente retorna:
🔴 Bloqueante | 🟡 Atenção | 🟢 Sugestão

    Agentes do projeto

    • .github/agents/qa-academia.agent.md — revisor de QA

    O @qa-academia verifica

    • Acessibilidade — aria-label, roles, contraste WCAG AA
    • SEO — meta tags, alt em imagens, headings
    • Performance — imagens sem lazy loading, scripts bloqueantes
    • HTML semântico — uso de section, article, nav
  • 🔌
    Pilar 04

    MCP

    Model Context Protocol — conecta o agente a ferramentas externas: APIs, docs ao vivo, sistema de arquivos.

    • Ferramentas externas
    • Fetch de docs
    • Filesystem
    // .vscode/mcp.json
{
  "servers": {
    "fetch": { ... },
    "filesystem": { ... },
    "precos-academia": { ... }
  }
}

    Servidores MCP configurados

    • fetch — busca qualquer URL pública em tempo real (docs WCAG, APIs)
    • filesystem — explora o projeto com busca semântica
    • precos-academia — servidor local com preços dos planos

    Como usar no chat

    • "Busque as diretrizes WCAG 2.1 e aplique ao nosso CSS"
    • "Quais arquivos CSS têm as variáveis de cor?" — sem indicar caminho
    • "Qual o preço do plano Premium?" — consulta o servidor local
  • 🪝
    Pilar 05

    Hooks

    Único primitivo que bloqueia e corrige o agente. Intercepta ações e injeta feedback automático.

    • Bloqueia ações
    • postToolUse
    • Lint automático
    Eventos disponíveis:
preToolUse  → único que BLOQUEIA antes
postToolUse → valida após edição ← nosso
sessionStart → log de início
userPromptSubmitted → filtra dados

    Hook configurado: lint-guard

    • .github/hooks/lint-guard.json — definição do hook
    • .github/hooks/scripts/lint-guard.ps1 — Windows
    • .github/hooks/scripts/lint-guard.sh — Linux/Mac

    Regras que o hook verifica

    • Proibido <div> onde elemento semântico se aplica
    • Toda <img> deve ter atributo alt
    • Classes CSS devem usar prefixo fc-
    • outline: none remove foco — violação WCAG 2.4.7

    Pilar 06

    Knowledge Base

    Documentação versionada no repo que o Copilot consulta e aplica diretamente no código.

    • Pesquisa semântica
    • Identidade visual
    • Tom de voz
    @workspace Qual é a cor primária da marca?
# → Copilot encontra em identidade-visual.md:
--fc-color-primary: #10b981

# Sem você copiar e colar nada.

    Wiki do repositório

    • docs/wiki/identidade-visual.md — paleta, tipografia, bordas
    • docs/wiki/conteudo-academia.md — tom de voz, frases proibidas
    • docs/wiki/suporte-sla.md — SLA de atendimento

    Por que importa

    • Preço do plano alterou? Atualiza o Markdown — o agente usa automaticamente
    • Nova cor na identidade? Um arquivo, zero retrabalho no CSS
    • Prompts dedicados em .github/prompts/
  • ⚙️
    Pilar 07

    Coding Agent

    Copilot autônomo no GitHub.com. Você atribui uma issue — ele explora, codifica e abre o PR automaticamente.

    • Issue → PR
    • Autônomo
    • AGENTS.md
    1. Cria issue no GitHub
2. Atribui ao Copilot no campo Assignees
3. Copilot lê AGENTS.md + instructions
4. Explora o repo, escreve HTML/CSS
5. Abre draft PR automaticamente

    Arquivo de controle

    • AGENTS.md — instrui o Coding Agent sobre regras do time
    • .github/workflows/deploy.yml — pipeline que ele respeita

    O AGENTS.md define

    • Princípio do fix mínimo — não refatora sem solicitação
    • Branches e fluxo de trabalho (feature/front, feature/conteudo…)
    • Arquivos que o agente pode ou não modificar
    • Padrão de commit Conventional Commits
  • 💻
    Pilar 08

    CLI

    GitHub Copilot direto no terminal — sugestões de comandos shell, explicações e agente interativo completo.

    • gh copilot
    • suggest / explain
    • Agente no terminal
    # Sugestão de comando
gh copilot suggest "reverter último commit"

# Explicação de comando
gh copilot explain "git rebase -i HEAD~3"

# Agente interativo completo
gh copilot -p "verifique alt em src/"

    Dois modos de uso

    • gh copilot suggest — sugere o comando shell correto em linguagem natural
    • gh copilot explain — explica qualquer comando git/shell
    • gh copilot — agente interativo: edita arquivos, roda lint, delega com &

    Pré-requisito

    • gh v2.93+ instalado — já disponível neste ambiente
    • O subcomando gh copilot faz download automático na primeira execução
    • Sem necessidade de instalar extensão separada

🚀 Primeiros passos para novos integrantes

Siga essa ordem para entender e validar todos os pilares.

  • 1

    Leia o PARTICIPANTE.md Visão geral do workshop e dos 8 pilares, com instruções passo a passo.

  • 2

    Abra o index.html localmente Use Live Server no VS Code e veja a página base da academia.

  • 3

    Inspecione .github/ Explore instructions, skills, agents e hooks — leia cada arquivo antes de usar.

  • 4

    Teste @qa-academia No Copilot Chat, invoque o agente para revisar qualquer arquivo HTML do projeto.

  • 5

    Rode gerar_evidencia.py Valida todos os pilares e gera o HTML de evidência com seu nome e turma.

  • 6

    Teste o CLI Execute gh copilot suggest e gh copilot explain no terminal.
